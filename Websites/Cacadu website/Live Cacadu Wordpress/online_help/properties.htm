Task Properties

Users of Cacadu can view and modify all critical properties of a task using the Task Properties Window. Cacadu helps organizing the screen real estate by allowing the user to dock the task properties in any corner of the application or ,if preferred, to float it separately.

 

The available task properties are listed below:

 

•Actions Count

 

  Displays the number of actions associated with a task.

 

•Break On Fail

 

  If true, aborts task's execution if any of the associated actions fail.

 

•May Fired Again

 

  Lets the user know if the task would execute again, as a result of the task being associated to triggers.

 

•Parent Group

 

  Displays the current group of the task and allows to modify it.

 

•Properties

 

  This property displays all the common properties of the task. Such as Task name, Enabled and Triggers When.

 

•Tasks variables

 

  Displays all the variables that have been used to configure the task.

 

•Triggers Count

 

  Displays the number of triggers associated with the task.

 

•Wait between actions

 

  It allows users to manage the delay time for the actions defined in a task.
