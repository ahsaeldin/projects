Navigation:  Actions > System >

Start Application

The first action and of one of the most important and main actions. It enables you to start an application along with other interesting options. Start using it by dragging/clicking from the Actions Gallery.

 

 

 

Before added by you, the next window will be open, asking you some questions to configure the action. This action's window has 4 main tabs:

 

•Simple: The first one and the simplest. just write the file name and click OK to quickly configure your action.

 

•Advanced: Has other advanced options that you may need to use in your action, like arguments to be passed as a command line parameters to the application, Working Directory, Window style and comments.

 

•Delay: Specify what to do after an action do its job.

 

•Errors: Specify what to do if an error occurs.

 

Note: Enabled check box enables you to disable/enable the action.

 

In this action, you can set File Name using different ways:

 

     1.Manually write application file name.

 

2.Write "<" and all tasks variables will be pulled down, so you can select one of them that holds your intended file name. Also you can directly write the variable name.

 

3.Write "G<" to do the same with Global Variables and "G{" to select from Global Constants

 

3.Aside from using variables and constants, you click the combo box arrow button to display all the applications defined under Path system environment variable.

 

4.Clicking the dotted button to browse for the selected application.

 

Note:You can click the reload button to refresh the combo box with constants, variables, and applications defined under the Path system environment variable.

 

The advanced tab of this action enables you to set additional properties that will extend the action's benefits...

 

•Verb: Will be loaded with the available file verbs like (open, open with, print).

 

•Arguments: To set the arguments/commands to be send to the application before loading.

 

•Working Directory: Set the working directory for the application.

 

•Window Style: Set the window style (has four values Hidden, Normal, Minimized and Maximized).

 

•You can right any comments about the action in the comments text box. The comments will be displayed beside action's name in the Task Flow.
